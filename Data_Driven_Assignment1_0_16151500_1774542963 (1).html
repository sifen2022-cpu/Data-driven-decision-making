Moltbook Social Network Analysis¶

Introduction

Moltbook is an experimental social platform designed for AI agents to type postings and replies, write and comment whic as part of their work flows than humans.It is modeled after Reddit which positioned as the central hub to the agent ecosystem.

This notebook examines Moltbook as a socil network dataset launched by enterpreneur Matt Schlicht in January 2026. Each data holds information about the main network analysis methods which include degree distribution analysis, connected components analysis, path analysis, clustering coefficient, centralyity analysis and how the emprical network is compared with (Erdos Renyi) ER, (Barabosi) BA, and (Watts-Strogatz) WS graph models.

Our aim is to understand using the above network analysis methods and to compare and prove which method has influence according to the three synthetic networks.

from itertools import combinations
import networkx as nx

MIN_AUTHORS_PER_SUBMOLT = 2
MAX_AUTHORS_PER_SUBMOLT = 1000
EXCLUDED_SUBMOLTS = set()

author_submolt = posts_df[["author_name", "submolt_name"]].drop_duplicates().copy()

submolt_author_groups = (
    author_submolt.groupby("submolt_name")["author_name"]
    .apply(list)
    .to_dict()
)

print("\n=== Number of submolts before filtering ===")
print(len(submolt_author_groups))

filtered_submolt_author_groups = {}

for submolt, authors in submolt_author_groups.items():
    unique_authors = list(set(authors))
    n_authors = len(unique_authors)

    if str(submolt).lower() in EXCLUDED_SUBMOLTS:
        continue
    if n_authors < MIN_AUTHORS_PER_SUBMOLT:
        continue
    if n_authors > MAX_AUTHORS_PER_SUBMOLT:
        continue # Corrected: Exclude submolts with too many authors

    # If none of the above conditions led to a 'continue', add to filtered groups
    filtered_submolt_author_groups[submolt] = unique_authors

print("\n=== Number of submolts after filtering ===")
print(len(filtered_submolt_author_groups))

G = nx.Graph()

for _, row in author_stats.iterrows():
    G.add_node(
        row["author_name"],
        total_posts=row["total_posts"],
        total_upvotes=row["total_upvotes"],
        total_downvotes=row["total_downvotes"],
        total_comments_received=row["total_comments_received"],
        avg_upvotes=row["avg_upvotes"],
        avg_comments_received=row["avg_comments_received"],
        primary_submolt=row["primary_submolt"],
    )

for submolt, authors in filtered_submolt_author_groups.items():
    for a1, a2 in combinations(sorted(authors), 2):
        if G.has_edge(a1, a2):
            G[a1][a2]["weight"] += 1
        else:
            G.add_edge(a1, a2, weight=1)

print("\n=== Network Overview ===")
print(f"Number of nodes:         {G.number_of_nodes():,}")
print(f"Number of edges:         {G.number_of_edges():,}")
print(f"Density:                 {nx.density(G):.6f}" if G.number_of_nodes() > 1 else "Density:                 0.000000")
print(f"Average clustering:      {nx.average_clustering(G):.6f}" if G.number_of_nodes() > 0 and G.number_of_edges() > 0 else "Average clustering:      0.000000")

isolates = list(nx.isolates(G))
G_main = G.copy()
G_main.remove_nodes_from(isolates)

print("\n=== Main Graph After Removing Isolates ===")
print(f"Nodes remaining:         {G_main.number_of_nodes():,}")
print(f"Edges remaining:         {G_main.number_of_edges():,}")
print(f"Removed isolates:        {len(isolates):,}")

def safe_density(graph):
    if graph.number_of_nodes() <= 1:
        return 0.0
    return nx.density(graph)

def safe_average_clustering(graph):
    if graph.number_of_nodes() == 0 or graph.number_of_edges() == 0:

def largest_component_subgraph(graph):
    if graph.number_of_nodes() == 0:
        return graph.copy()
    if graph.number_of_edges() == 0:
        first_node = list(graph.nodes())[0]
        return graph.subgraph([first_node]).copy()
    comps = list(nx.connected_components(graph))
    return graph.subgraph(max(comps, key=len)).copy()

def summarize_graph(name, graph):
    lcc = largest_component_subgraph(graph)

    if lcc.number_of_nodes() >= 2 and lcc.number_of_edges() > 0:
        avg_shortest_path = nx.average_shortest_path_length(lcc)
        diameter = nx.diameter(lcc)
    elif lcc.number_of_nodes() == 1:
        avg_shortest_path = 0.0
        diameter = 0
    else:
        avg_shortest_path = np.nan
        diameter = np.nan

    degrees_local = [d for _, d in graph.degree()]
    avg_degree = np.mean(degrees_local) if len(degrees_local) > 0 else 0.0
    max_degree = np.max(degrees_local) if len(degrees_local) > 0 else 0.0

    return {
        "graph": name,
        "nodes": graph.number_of_nodes(),
        "edges": graph.number_of_edges(),
        "density": safe_density(graph),
        "avg_clustering": safe_average_clustering(graph),
        "largest_component_size": lcc.number_of_nodes(),
        "avg_shortest_path": avg_shortest_path,
        "diameter": diameter,
        "avg_degree": avg_degree,
        "max_degree": max_degree,
    }

Conclusion¶

This project analyzed Moltbook as an author network, where nodes represent authors and edges represent shared participation in the same submolt. After cleaning the data, the final network contained 1,456 connected authors and 164,589 edges, so it met the assignment requirement for a large enough network. The results showed that the network is very dense and highly clustered, which suggests that Moltbook authors tend to form tight groups around shared interests.

The path analysis showed that most authors are only a few steps away from each other, meaning the network is still quite connected despite its size. The centrality results also showed that a small number of authors are much more important than others, either because they are highly connected or because they act as bridges between different parts of the network.

When compared with the ER, BA, and WS models, the Moltbook network stood out for having much stronger clustering than all three. This suggests that the platform is not random, but is shaped by community structure. The community detection results also showed that Moltbook has clear groups, although they are not completely separated from one another.

Overall, the findings suggest that Moltbook is a socially cohesive platform with strong local communities, a few important bridge users, and moderate fragmentation. One limitation is that the network was built from shared submolt participation rather than direct replies, so it reflects common activity patterns more than direct user-to-user interaction.
